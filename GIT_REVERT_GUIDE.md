## צפייה בהיסטוריה של הפרויקט

### הצגת רשימת הקומיטים
```bash
# הצגת כל הקומיטים עם מזהה מקוצר והודעה
git log --oneline

# הצגת היסטוריה מפורטת יותר
git log

# הצגת היסטוריה עם גרף
git log --oneline --graph
```

### חיפוש קומיט לפי תאריך
```bash
# הצגת קומיטים מתאריך מסוים
git log --since="2024-01-01"

# הצגת קומיטים לפני תאריך מסוים
git log --until="2024-06-01"

# הצגת קומיטים בטווח תאריכים
git log --since="2024-01-01" --until="2024-06-01"
```

## חזרה זמנית לקומיט קודם

### להסתכל על מצב קודם (בלי לשנות דבר)
```bash
# החלף <commit-hash> במזהה הקומיט שרוצים לראות
git checkout <commit-hash>

# לחזור למצב הנוכחי
git checkout main
# או
git checkout master
```

**שים לב:** פקודה זו מעבירה אותך למצב "detached HEAD" - אתה יכול להסתכל ולבדוק, אבל שינויים לא יישמרו אלא אם תיצור branch חדש באמצעות `git switch -c <branch-name>` או `git checkout -b <branch-name>`.

## חזרה קבועה לקומיט קודם

### אפשרות 1: Git Revert (מומלץ - בטוח)
יוצר קומיט חדש שמבטל את השינויים, משמר את ההיסטוריה.

```bash
# ביטול קומיט אחד ספציפי
git revert <commit-hash>

# ביטול מספר קומיטים (יש לציין כל קומיט בנפרד)
git revert <commit1> <commit2> <commit3>

# ביטול ללא יצירת קומיט אוטומטי
git revert --no-commit <commit-hash>
```

### אפשרות 2: Git Reset (זהירות!)
מחזיר את הענף לקומיט קודם. **יכול למחוק היסטוריה!**

```bash
# שמירת השינויים כ-uncommitted (בטוח יותר)
git reset --soft <commit-hash>

# מחיקת השינויים מה-staging area
git reset --mixed <commit-hash>

# מחיקה מוחלטת של כל השינויים (זהירות!)
git reset --hard <commit-hash>
```

**אזהרה:** `git reset --hard` מוחק שינויים לצמיתות!

## ביטול שינויים ספציפיים

### ביטול שינויים בקובץ שלא נשמרו עדיין
```bash
# ביטול שינויים בקובץ ספציפי (פקודה מודרנית - מומלץ)
git restore <filename>

# ביטול כל השינויים שלא נשמרו
git restore .

# פקודה ישנה יותר (עדיין עובדת)
git checkout -- <filename>
```

### ביטול קובץ שכבר הוסף ל-staging
```bash
# הסרה מה-staging area (פקודה מודרנית - מומלץ)
git restore --staged <filename>

# הסרת כל הקבצים מה-staging
git restore --staged .

# פקודה ישנה יותר (עדיין עובדת)
git reset HEAD <filename>
```

## שחזור קבצים בודדים

### שחזור קובץ ממצב קודם
```bash
# שחזור קובץ מקומיט ספציפי
git checkout <commit-hash> -- <filename>

# שחזור קובץ מהקומיט הקודם
git checkout HEAD~1 -- <filename>

# שחזור כל הקבצים מקומיט ספציפי
git checkout <commit-hash> -- .
```

## דוגמאות מעשיות

### דוגמה 1: חזרה לפני המיזוג האחרון
```bash
# הצגת היסטוריה כדי למצוא את הקומיט לפני המיזוג
git log --oneline

# ביטול המיזוג האחרון
git revert -m 1 HEAD
```

### דוגמה 2: חזרה לגרסה ישנה לגמרי
```bash
# מציאת הקומיט הרצוי
git log --oneline --since="2023-01-01" --until="2023-12-31"

# יצירת branch חדש מהמצב הישן (פקודה מודרנית)
git switch -c old-version <commit-hash>

# או באמצעות הפקודה הישנה
git checkout -b old-version <commit-hash>
```

### דוגמה 3: ביטול קומיט ספציפי באמצע ההיסטוריה
```bash
# ביטול קומיט ספציפי (יוצר קומיט הפוך)
git revert <commit-hash>
```

## טיפים חשובים

1. **תמיד עשה גיבוי** לפני פעולות שמשנות היסטוריה
2. **השתמש ב-`git status`** לפני ואחרי כל פעולה
3. **העדף `git revert`** על פני `git reset` בפרויקטים משותפים
4. **בדוק עם `git log`** שאתה במקום הנכון
5. **יצור branch חדש** לפני ניסויים מסוכנים

## פקודות שימושיות נוספות

```bash
# הצגת מה השתנה בקומיט ספציפי
git show <commit-hash>

# הצגת ההבדלים בין שני קומיטים
git diff <commit1>..<commit2>

# הצגת מי שינה כל שורה בקובץ
git blame <filename>

# חיפוש בהיסטוריה לפי הודעת קומיט
git log --grep="keyword"
```
